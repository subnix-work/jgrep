🔍 What is jgrep?

The Concept

jgrep is grep for JSON and YAML data. Instead of fumbling with raw text patterns, you use jq filters to precisely query fields, filter objects, and search arrays.

Perfect for log analysis, API responses, Kubernetes manifests, config files — anything structured.

How it works

jgrep reads JSON or YAML from stdin or files / directories, applies your jq filter, and outputs only the matches — one match per line.

Also supports Newline-delimited JSON (NDJSON): multiple JSON objects per file, one per line.

Why not just jq?

jq transforms — jgrep greps. Exit code 0 = match found, exit code 1 = no match, just like you'd expect from grep.

Add flags like -l (filenames only), -c (count), and -r (recursive) — your grep muscle memory works here.

One liner

cat logs.json | jgrep '.[] | select(.level == "ERROR")'

That's it. All error entries from your JSON log — no AWK workarounds, no grepping raw JSON text.

⚡ Usage

Syntax: jgrep [OPTIONS] FILTER [FILE...]
Invocation basic
# Read from stdin
cat data.json | jgrep '.name'

# Pass a file directly
jgrep '.name' data.json

# Multiple files
jgrep '.status' *.json

# Recursively search a directory
jgrep -r '.name' ./logs/

🎯 Filtering with select()

select(CONDITION) returns only the objects for which the condition is true. That's the core of jgrep.

Only entries from Berlin filter
Command
cat example.json | jgrep \
  '.[] | select(.city == "Berlin")'
Output
{"id":1,"name":"Alice",...,"city":"Berlin",...}
{"id":3,"name":"Clara",...,"city":"Berlin",...}
Only active users filter
Command
cat example.json | jgrep \
  '.[] | select(.active == true)'
Output
{"id":1,"name":"Alice",...,"active":true,...}
{"id":3,"name":"Clara",...,"active":true,...}
Only inactive users filter
Command
cat example.json | jgrep \
  '.[] | select(.active == false)'
Output
{"id":2,"name":"Bob",...,"active":false,...}
Numeric comparison: age > 28 filter
Command
cat example.json | jgrep \
  '.[] | select(.age > 28)'
Output
{"id":1,"name":"Alice","age":30,...}
{"id":3,"name":"Clara","age":35,...}
Age in a range: 25–30 filter
Command
cat example.json | jgrep \
  '.[] | select(.age >= 25 and .age <= 30)'
Output
{"id":1,"name":"Alice","age":30,...}
{"id":2,"name":"Bob","age":25,...}
OR condition: Berlin or Hamburg filter
Command
cat example.json | jgrep \
  '.[] | select(.city == "Berlin" or .city == "Hamburg")'
Output
{"id":1,...,"city":"Berlin",...}
{"id":2,...,"city":"Hamburg",...}
{"id":3,...,"city":"Berlin",...}
Only the name of matched objects filter
Command
cat example.json | jgrep \
  '.[] | select(.city == "Berlin") | .name'
Output
Alice
Clara

🌿 Nested Objects

Deeper JSON structures — like API responses or logs — are navigated with simple dot notation.

Example structure (nested.json)
[
  {
    "user": { "id": 1, "name": "Alice" },
    "address": { "city": "Berlin", "zip": "10115" },
    "scores": [95, 87, 92]
  },
  {
    "user": { "id": 2, "name": "Bob" },
    "address": { "city": "Hamburg", "zip": "20095" },
    "scores": [70, 80]
  }
]
Read a deeply nested field basic
Command
cat nested.json | jgrep \
  '.[].user.name'
Output
Alice
Bob
Filter by city in a nested object filter
Command
cat nested.json | jgrep \
  '.[] | select(.address.city == "Berlin")'
Output
{"user":{"id":1,"name":"Alice"},"address":{"city":"Berlin",...},...}
Filter by a high score in an array filter
Command
cat nested.json | jgrep \
  '.[] | select(.scores[] > 90)'
Output
{"user":{"id":1,"name":"Alice"},...,"scores":[95,87,92]}
Only the user name from Berlin entries filter
Command
cat nested.json | jgrep \
  '.[] | select(.address.city == "Berlin") | .user.name'
Output
Alice

🚩 Flags & Options

Flag Long form Description
-r --recursive Recurse into directories and search all .json files
-l --files-with-matches Only print filenames that contain matches
-c --count Print match count per file instead of the values
-s --slurp Collect all matching results (across all files) into a single JSON array
-n --null-input Use null as input — evaluate the filter without reading any file
-f --from-file=FILE Read the filter expression from a .jq file instead of the command line
--pretty Pretty-print JSON output with indentation
--yaml Read input as YAML; .yml and .yaml files are detected automatically
--color-level Color each output line by log level (log.level, level, severity, ...)
--color-level-field FIELD Use a custom field for --color-level, for example app.level
--no-color Disable syntax-highlighted output (also respects $NO_COLOR)
-h --help Show help message
--version Print version number

📂 Multiple Files & Pipelines

Multiple files at once basic
Command
jgrep '.name' users.json admins.json
Output
users.json: Alice
users.json: Bob
admins.json: Root
Inside a shell pipeline basic
# Pipe an API response directly
curl -s https://api.example.com/users | jgrep '.[] | select(.role == "admin")'

# Filter logs and count
cat app.log.json | jgrep 'select(.level == "ERROR")' | wc -l

# Combine with grep
jgrep '.[] | .name' users.json | grep "Ali"

# Write results to a file
cat events.json | jgrep 'select(.type == "click")' > clicks.json
NDJSON — one JSON object per line basic
logs.ndjson
{"level":"INFO","msg":"started"}
{"level":"ERROR","msg":"connection refused"}
{"level":"INFO","msg":"retrying"}
{"level":"ERROR","msg":"timeout"}
Command + Output
cat logs.ndjson | jgrep \
  'select(.level == "ERROR")'

{"level":"ERROR","msg":"connection refused"}
{"level":"ERROR","msg":"timeout"}
YAML files work with the same jq filters yaml
deployment.yaml
apiVersion: apps/v1

  name: checkout

  replicas: 3
  template:

kubectl get deploy checkout -o yaml | jgrep --yaml \

🧾 Kubernetes / ECS Logs

Kubernetes containers and AWS ECS often write one JSON object per line. jgrep reads that NDJSON directly, so you can filter by structured fields first and then project the match into a readable text line.
ECS-style log input ndjson
{"@timestamp":"2026-06-28T08:15:30Z","log.level":"INFO","message":"payment accepted","service.name":"checkout","kubernetes":{"namespace":"shop","pod":"checkout-7b9"},"details":{"order_id":"A-1001","amount":42.5},"additions":{"tenant":"eu"},"custom_tracker":{"trace_id":"tr-91"}}
{"@timestamp":"2026-06-28T08:16:02Z","log.level":"WARN","message":"slow downstream response","service.name":"checkout","kubernetes":{"namespace":"shop","pod":"checkout-7b9"},"details":{"dependency":"inventory","latency_ms":850},"additions":{"tenant":"eu"},"custom_tracker":{"trace_id":"tr-92"}}
{"@timestamp":"2026-06-28T08:16:12Z","log.level":"ERROR","message":"payment declined","service.name":"checkout","kubernetes":{"namespace":"shop","pod":"checkout-7b9"},"details":{"order_id":"A-1002","reason":"card_expired"},"additions":{"tenant":"eu"},"custom_tracker":{"trace_id":"tr-93"}}
JSON to human-readable text text
Command
cat ecs.ndjson | jgrep \
  '"\(.["@timestamp"]) [\(.["log.level"])] \(.["service.name"])/\(.kubernetes.pod): \(.message)"'
Output without JSON braces
2026-06-28T08:15:30Z [INFO] checkout/checkout-7b9: payment accepted
2026-06-28T08:16:02Z [WARN] checkout/checkout-7b9: slow downstream response
2026-06-28T08:16:12Z [ERROR] checkout/checkout-7b9: payment declined
Filter objects, then print text filter
Command
cat ecs.ndjson | jgrep \
  'select(.["log.level"] == "ERROR" and .kubernetes.namespace == "shop") |
   "\(.["@timestamp"]) ERROR \(.["service.name"]): \(.message) trace=\(.custom_tracker.trace_id)"'
Output
2026-06-28T08:16:12Z ERROR checkout: payment declined trace=tr-93
Unpack details / additions / custom_tracker advanced
Command
cat ecs.ndjson | jgrep \
  'select(.details.reason? or .details.latency_ms?) |
   "\(.["@timestamp"]) [\(.["log.level"])] \(.message) tenant=\(.additions.tenant) trace=\(.custom_tracker.trace_id) details=\(.details)"'
Output
2026-06-28T08:16:02Z [WARN] slow downstream response tenant=eu trace=tr-92 details={"dependency":"inventory","latency_ms":850}
2026-06-28T08:16:12Z [ERROR] payment declined tenant=eu trace=tr-93 details={"order_id":"A-1002","reason":"card_expired"}
Color whole lines by log level flag
Command
cat ecs.ndjson | jgrep --color-level \
  '"[\(.["log.level"])] \(.["@timestamp"]) \(.message) trace=\(.custom_tracker.trace_id)"'
Visual result
[INFO]  2026-06-28T08:15:30Z payment accepted trace=tr-91
[WARN]  2026-06-28T08:16:02Z slow downstream response trace=tr-92
[ERROR] 2026-06-28T08:16:12Z payment declined trace=tr-93
Custom log level field flag
Command
cat app.ndjson | jgrep --color-level \
  --color-level-field app.level \
  '"[\(.app.level)] \(.message)"'
Output
[WARN] slow downstream response
Kubernetes pod logs pipeline
# Filter live logs from kubectl, keep only checkout errors, print readable lines
kubectl logs -n shop deploy/checkout --since=10m | jgrep --color-level \
  'select(.["log.level"] == "ERROR") |
   "\(.["@timestamp"]) \(.kubernetes.pod // "-") \(.message) order=\(.details.order_id // "-")"'

🧠 Advanced Filters

Regex: names starting with A advanced
Command
cat example.json | jgrep \
  '.[] | select(.name | test("^A"))'
Output
{"id":1,"name":"Alice",...}
Regex: names containing "a" (case-insensitive) advanced
Command
cat example.json | jgrep \
  '.[] | select(.name | test("[Aa]"; "i"))'
Output
{"id":1,"name":"Alice",...}
{"id":3,"name":"Clara",...}
has() — check if a field exists advanced
Command
cat example.json | jgrep \
  '.[] | select(has("tags"))'
Output (all entries that have a "tags" field)
{"id":1,...}
{"id":2,...}
{"id":3,...}
Project specific fields only advanced
Command
cat example.json | jgrep \
  '.[] | {name: .name, city: .city}'
Output
{"name":"Alice","city":"Berlin"}
{"name":"Bob","city":"Hamburg"}
{"name":"Clara","city":"Berlin"}
Combined: Berlin developers with field projection advanced
Command
cat example.json | jgrep \
  '.[] | select(.city == "Berlin" and (.tags[] == "developer")) | {name: .name, age: .age}'
Output
{"name":"Alice","age":30}
{"name":"Clara","age":35}
length — filter by array size advanced
Command
cat example.json | jgrep \
  '.[] | select((.tags | length) > 1) | .name'
Output (people with more than one tag)
Alice
Bob
Clara
type — filter by value type advanced
Command
# Only output fields whose value is a number
cat example.json | jgrep \
  '.[] | .age | select(type == "number")'
Output
30
25
35

Command
# Generate data without any input file

# Simple arithmetic

Output

-f / --from-file — load filter from a .jq file flag
filter.jq
.[]
| select(.level == "ERROR" and (.msg | test("timeout")))
| {ts: .timestamp, msg: .msg}
Command
jgrep -f filter.jq logs.ndjson
Syntax highlighting — automatic when writing to a terminal flag

jgrep '.[] | select(.active)' users.json
# → keys are cyan, strings green, numbers yellow, booleans/null magenta

# Disable with --no-color or $NO_COLOR env variable

📖 Cheatsheet

Fields
.name              # read a field
.user.name         # nested field
.tags[0]           # array index
.tags[-1]          # last element
.[]                # iterate all values
.tags[]            # all array elements
select() — Conditions
select(.x == "val")       # equal
select(.x != "val")       # not equal
select(.n > 10)           # greater than
select(.n >= 10)          # greater or equal
select(.a and .b)         # AND
select(.a or .b)          # OR
select(.a | not)          # NOT
String Functions
select(.s | test("regex"))     # regex match
select(.s | startswith("A"))   # prefix
select(.s | endswith("z"))     # suffix
select(.s | contains("sub"))   # substring
.name | ascii_downcase         # lowercase
.name | ltrimstr("Mr.")        # trim prefix
Array Functions
select(.arr | length > 2)          # length
select(.arr[] == "x")              # find element
select(.arr | contains(["x"]))    # contains
select(.arr | any(. > 5))         # at least one
select(.arr | all(. > 0))         # all match
.arr | first                       # first element
Pipes & Projection
.[] | select(.x) | .y     # filter + project
{name: .name, id: .id}    # build new object
[.[] | .name]             # array of names
.[] | .x, .y             # multiple fields
Flags
-r         # recursive (directory)
-l         # filenames with matches only
-c         # count matches
-s         # slurp — collect into array
-n         # null-input (no file needed)
-f FILE    # read filter from .jq file
--pretty   # pretty-print JSON output
--yaml     # parse stdin as YAML
--color-level # color log lines by level
--no-color # disable color output

🔢 Exit Codes

0
At least one match found — just like grep
1
No match — filter returned nothing
2
Error — invalid filter or JSON parse failure
# Use in shell scripts
jgrep '.[] | select(.active == true)' users.json
if [ $? -eq 0 ]; then
  echo "Active users found"
else
  echo "No active users"
fi
